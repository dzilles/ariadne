# Ariadne Implementation Plan

This document serves as the roadmap for the autonomous V-Model architecture.

## Phase 1: Core Infrastructure - ✅ COMPLETED
- [x] **Remove Old Guard:** Deleted `activate_ticket` logic.
- [x] **Build Backend Context:** Implemented global `active_ticket_id` storage.
- [x] **Implement JIT Guard:** Created `@jit_vmodel_guard` with status-based rule checking.
- [x] **Plane Robustness:** Implemented "State Enrichment" and "ID Resolution" (UUID vs Seq ID).
- [x] **Done State Unblocking:** Defined rules for terminal states to allow reopening/commenting.

## 🛠️ Ariadne Refinement TODO (V-Model V2)

### 1. The "Done" State Unblocking - ✅ DONE
*   **Issue:** Agents were deadlocked once a ticket hit Done.
*   **Fix:** Updated `src/workflows/rules.py` to allow `update_status`, `post_comment`, and `read_file` in the Done state.

### 2. ID Resolution Layer - ✅ DONE
*   **Issue:** Constant 404s due to Seq ID vs UUID mismatch.
*   **Fix:** Added `_resolve_id` to `PlaneInteraction`. Every tool now automatically resolves numbers to UUIDs.

### 3. Flexible Surgical Delegation - ✅ DONE
*   **Issue:** Agents either wander too much or are too restricted.
*   **Fix:** Refine Orchestrator and Agent prompts to use "Primary Manifests."
*   **Suggestion:** Instead of "Only read these files," the prompt should say: "Your primary context is in [Files]. You should prioritize these. If you identify a dependency outside this list, you may explore it, but you MUST state your reasoning in your 'Thought'."

### 4. Dynamic ID Discovery (No JSON Registry) - ✅ DONE
*   **Issue:** We need Stable IDs without a brittle central registry file.
*   **Fix:** Implement "Discovery-by-Search."
*   **Steps:**
    *   **Search:** Requirements Agent uses `search_file_content` for keywords in `docs/requirements/` before creating new ones.
    *   **Allocation:** Agent finds the highest existing number (e.g., `grep -r "REQ-" docs/`) and increments it.
    *   **Linkage:** Use a Plane Label (e.g., `sid:REQ-001`) as the searchable bridge.

### 5. Code-Level Traceability - ✅ DONE
*   **Issue:** Missing links in source code.
*   **Fix:** Mandatory docstring tags.
*   **Suggestion:** Update Developer Agent's system prompt: "When modifying functions/classes, you MUST include a traceability tag: `Implementation of ARCH-XXX. Fulfills REQ-XXX.`"

### 6. JIT Guard "State Enrichment" - ✅ DONE
*   **Issue:** Guard failed when Plane returned `null` for state objects in lists.
*   **Fix:** `PlaneInteraction` now performs a "deep fetch" by UUID to populate `state_detail`.
